# Job Description Database Setup

This document tracks the changes made to the database schema to support the **JD Parsing Agent**.

## 🚀 Changes Summary
We expanded the initial `jobs` table to become a more robust `job_description` table. This allows us to store both the high-level metadata (Company, Salary, Location) and the complex structured data generated by the AI agent (Skills Rubric).

## 📂 File Mapping
| Component | File Path | Description |
|-----------|-----------|-------------|
| **SQLAlchemy Model** | `backend/app/db/models.py` | Defines the `JobDescription` class and its column types. |
| **Package Export** | `backend/app/db/__init__.py` | Exports `JobDescription` so it can be imported cleanly across the app. |
| **Database Migration**| `backend/alembic/versions/0002_update_jobs_table.py` | The script that renamed the table and added the new columns. |

## 📊 Schema Details (`job_description` table)

### Renames
- **Table Name**: `jobs` → `job_description`
- **Rubric Column**: `rubric` → `structured_output` (Stores the Pydantic `JDRubric` model).

### New Metadata Columns
- `experience_level` (Float): Required years of experience.
- `company_name` (String): The hiring company.
- `location` (String): Office location or Remote status.
- `employment_type` (String): Full-time, Contract, Freelance, etc.
- `salary_range` (Float): Extracted numerical salary data.
- `source` (String): Where the JD was retrieved from.

### New Agent Audit Columns
- `confidence_score` (Float): How confident the LLM is in the extracted data.
- `parsed_at` (DateTime): When the agent finished processing.
- `error_message` (Text): Stores logs if the agent fails.
- `raw_output` (JSONB): A "bucket" for any extra data the LLM provides that doesn't fit the schema yet.
